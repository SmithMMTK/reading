คุณสามารถบังคับใช้ความปลอดภัยของโมเดลข้อมูล (_model security_) ได้โดยการจำกัดการเข้าถึงข้อมูลบางส่วน และจำกัดการเข้าถึง _tables_ หรือ _columns_ เฉพาะภายในโมเดล คุณอาจต้องจำกัดการเข้าถึง เพราะผู้ใช้งานบางรายไม่ควรเห็นข้อมูลบางอย่าง เช่น ผลการขายของภูมิภาคอื่น

วิธีที่ใช้กันทั่วไปในการตอบสนองความต้องการนี้คือการตั้งค่า **Row-Level Security (RLS)** โดยกำหนด **roles** และ **rules** ที่กรองข้อมูลภายในโมเดล และคุณยังสามารถตั้งค่า **Object-Level Security (OLS)** เพื่อจำกัดการเข้าถึงทั้งตารางหรือคอลัมน์ได้ด้วย

## Learning objectives

เมื่อจบโมดูลนี้ คุณจะสามารถ:

- จำกัดการเข้าถึงข้อมูลใน _Power BI model_ ด้วย _RLS_
- จำกัดการเข้าถึงวัตถุใน _Power BI model_ ด้วย _OLS_
- ใช้แนวปฏิบัติที่ดีในการพัฒนาเพื่อบังคับใช้ความปลอดภัยของ _Power BI model_
